- [x] Processing-hang / video-viewing / 3D-render fixes (2026-07-19, from live rig use):
  - **The big one — processing hung the ENTIRE backend, and that's why video wouldn't play either.** Root cause found: `Pose2Sim/filtering.py` forces the interactive `qtagg` matplotlib backend and creates a figure *at module-import time*; on Windows that call **hangs forever** the first time it runs on a worker thread (Qt GUI objects must be built on the main thread), and processing runs on a per-session worker thread. The hang holds the GIL, so every endpoint — including plain video serving — froze. Verified live: reprocessing a previously-stuck session now completes without hanging, and the server + video endpoint stay responsive (200/206) throughout.
